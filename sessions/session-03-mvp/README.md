# Session 3 — Ship Your MVP in a Weekend

> *"A 48-hour MVP isn't 'I built it fast'. It's 'I built it on purpose — and I know what I cut.'"*

**Series:** Vibe Coding Mastery — Mindset Foundations (Session 3 of 5)
**Status:** ⏳ In production
**Runtime:** 30 minutes (target)
**Cost:** Free — like every session in this series

## The promise

A repeatable 48-hour idea → live product framework. Not a list of tools. A way to slice an idea into a Friday-evening commit, a Saturday-night ship, and a Sunday-night first user.

## The trap discussed

Building features no user asked for. AI makes you fast, which makes this trap deadlier — you can ship 40 features over a weekend and discover none of them are the one that earns a customer.

## What you walk away with

- The **48-hour MVP slicer** — three columns: must-ship, should-ship, never-ship
- A **landing-before-product** template that runs in parallel to the build
- A **single-loop user test** — get one human to use it on Sunday before midnight
- New doctrine entries covering the patterns from your first PURE-loop weekend
